## 根本的ボトルネック

### 1. WiFiスキャンがIMUを殺す (最重要)
`scanNetworks()` は同期ブロック。スキャン中IMU完全停止。2秒に1回、最大数秒のデッドタイム発生。

**Fix案:**
- スキャン間隔を3000ms以上に伸ばし、Processing側の予測精度を上げる
- または UNO R4のWiFiS3に非同期スキャンAPIがあれば使う（要調査）

### 2. ヨー積分ドリフト無補正
`heading += lastGz * dt` のみ。加速度で姿勢補正なし。数十秒でズレる。

**Fix案 (短期):** 静止時にgz=0強制（現在の`moving==0`判定活用）  
**Fix案 (長期):** 地磁気センサ追加（MPU-6050にはない）→ HMC5883L等

### 3. 速度積分モデルが粗い
`vx += filtAx * ACCEL_GAIN * dt` → 加速度二重積分は誤差爆発しやすい。`WALK_SPEED`への強制ブレンドで誤魔化してるが、旋回中に破綻。

## 改善機会リスト (優先順)

### 🔴 高優先 — 精度・応答性直結

**A. 歩行検出 (PDR: Pedestrian Dead Reckoning)**  
加速度垂直成分のピーク検出で「1歩」をカウント。1歩 = 固定ストライド長(~0.65m)。連続積分より10倍安定。

```
歩行周期 = az のピーク間隔
ストライド = f(cadence, 身長)
位置更新 = heading方向 × ストライド
```

**B. IMUサンプリング高速化**  
現在50ms(20Hz)。MPU-6050は1kHzまで対応。`IMU_INTERVAL_MS = 10`(100Hz)で旋回追従が劇的に改善。ArduinoのシリアルボトルネックはJSON圧縮で対処済みなので問題少ない。

**C. 静止時ヨー補正**  
`moving==0` 時、`gzBias`をリアルタイム更新（低周波LPF）。ドリフト抑制。

```cpp
if (moving == 0) {
  gzBias = gzBias * 0.995f + gz * 0.005f;
}
```

**D. WiFi三角測位 (複数APで位置推定)**  
現在：AP1つごとに距離リング→ランダム角配置。  
改善：見えてるAP全部使って最小二乗法で自分の位置推定 → IMUドリフトのアンカーに使える。

```
(x-xi)² + (y-yi)² = di²  (i = 各AP)
過剰決定系 → 最小二乗解
```

**E. 持ってるHY-SRF05を使う**  
壁検出でスリップ/位置補正。「壁から0.5m」という制約がIMU誤差を殺せる。

---

### 🟡 中優先 — 精度向上

**F. RSSIフィルタリング (Arduino側)**  
同一APを複数回サンプリングして中央値 or 指数移動平均してから送信。生RSSIは±5dBmのノイズ。

**G. AP位置推定をベイズ更新に**  
現在のlerpより、各APの不確かさ（分散）を管理して観測更新するほうがAP位置収束が速い。

**H. チャンネル別パスロスモデル**  
2.4GHz(ch1-13)と5GHz(ch36+)で減衰特性違う。`rssiToDistance()`を周波数対応に。

```java
float rssiToDistance(int rssi, int channel) {
  float n = (channel > 14) ? 2.0f : 2.5f; // 5GHz直進性高い
  float A = (channel > 14) ? -56.0f : -59.0f;
  return pow(10.0f, (A - rssi) / (10.0f * n));
}
```

**I. カルマンフィルタ (IMU+WiFi融合)**  
状態: [x, y, vx, vy, heading]  
観測: WiFi位置推定 (低頻度・大ノイズ) + IMU (高頻度・ドリフトあり)  
EKF or UKFで最適融合。実装コストは高いが精度は飛躍的に上がる。

---

### 🟢 低優先 — QoL・拡張性

**J. マップ保存/再ロード**  
AP位置・経路をJSONで保存。次回起動時ロード→初期収束が速くなる。

**K. フィンガープリント法**  
特定地点のRSSIベクトルを記録→後でk-NNで「今どこ?」を推定。WiFiのみで位置わかる。

**L. UNO R4のLEDマトリクス活用**  
スキャン中/移動中/静止 をLED表示→シリアル開けなくてもデバッグ可。

**M. 複数センサ同時読み取り**  
MPU-6050のFIFOバッファ有効化→スキャン中にFIFOにIMUデータ貯めて、スキャン後一括送信でデッドタイム補間。
